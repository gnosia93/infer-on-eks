EKS 클러스터에서 초고속 네트워크인 EFA(Elastic Fabric Adapter) 트래픽을 모니터링하고 그라파나(Grafana)와 연동하는 것은 분산 학습이나 LLM 분산 추론(Ray, LWS 등)의 병목을 잡기 위해 매우 필수적인 작업입니다.
일반적인 네트워크 카드(ENA)와 달리, EFA는 커널을 우회하여 통신하므로 일반 Node Exporter로는 트래픽이 잡히지 않습니다. 따라서 AWS가 제공하는 전용 수집기를 배포해야 합니다. 전체적인 구현 아키텍처와 흐름을 정리해 드립니다.

### 🛠️ EFA 모니터링 구축 아키텍처 ###
전체적인 메커니즘은 다음과 같습니다.
* EFA Node Exporter 배포 (EKS DaemonSet): 각 GPU 노드 내부의 /sys/class/infiniband 경로에서 EFA 상세 카운터 메트릭을 수집하여 포트로 노출합니다.
* Prometheus 가 스크랩(Scrape): 노출된 메트릭을 Prometheus(또는 Amazon Managed Prometheus)가 가져가 저장합니다.
* Grafana 시각화: 그라파나 대시보드에서 쿼리하여 모니터링합니다.

### 1단계: EKS 클러스터에 EFA용 Exporter 배포 ###
AWS 샘플 진영에서 관리하는 **efa-node-exporter**를 EKS 클러스터에 데몬셋(DaemonSet) 형태로 배포해야 합니다. 이 엑스포터는 GPU 노드마다 하나씩 뜨며 EFA 카드 상태를 긁어옵니다.
* GitHub 공식 소스: aws-samples/awsome-distributed-training 내의 efa-node-exporter 이용.
* 배포 방식: 쿠버네티스 DaemonSet 매니페스트를 작성하여 배포합니다. 호스트의 네트워크와 통신해야 하므로 hostNetwork: true 설정과 /sys 파일시스템 볼륨 마운트가 필요합니다.

```
# efa-node-exporter-daemonset.yaml (간략화된 예시)
apiVersion: apps/v1
kind: DaemonSet
metadata:
  name: efa-node-exporter
  namespace: monitoring
spec:
  selector:
    matchLabels:
      app: efa-node-exporter
  template:
    metadata:
      labels:
        app: efa-node-exporter
    spec:
      hostNetwork: true
      hostPID: true
      containers:
      - name: efa-exporter
        image: public.ecr.aws/hpc-cloud/efa-node-exporter:latest # AWS 공개 이미지
        securityContext:
          privileged: true
        ports:
        - containerPort: 9100 # 메트릭 노출 포트
          name: metrics
        volumeMounts:
        - name: sys
          mountPath: /sys
          readOnly: true
      volumes:
      - name: sys
        hostPath:
          path: /sys
```

### 2단계: Prometheus 수집 설정 (Scrape Config) ###
배포된 efa-node-exporter가 뱉어내는 메트릭을 Prometheus가 긁어갈 수 있도록 PodMonitor 또는 Prometheus 설정을 추가합니다.
*	기본적으로 node-exporter와 유사한 방식으로 9100번 포트(또는 지정 포트)의 /metrics 엔드포인트를 타겟으로 잡습니다.

### 3단계: 어떤 메트릭을 봐야 할까? (핵심 지표) ###
그라파나 보드를 구성할 때, 단순히 트래픽 총량 외에 AWS가 최근 추가한 고도화된 EFA 손실 지표들을 꼭 모니터링해야 분산 추론 시 병목을 잡을 수 있습니다.
*	rdma_read_bytes / rdma_write_bytes: EFA를 통해 나아가고 들어온 실제 RDMA 데이터 처리량(Throughput).
*	tx_bytes / rx_bytes: 전체 전송/수신 바이트.
* retransmit_bytes / retransmit_pkts: 재전송된 패킷/바이트. 이 수치가 튀면 네트워크에 정체(Congestion)가 발생했거나 패킷 드랍이 일어나고 있다는 뜻입니다.
*	rx_dropped: 수신 버퍼 부족 등으로 드랍된 패킷 수.

### 4단계: Grafana 대시보드 연동 ###
그라파나 랩(Grafana Labs) 공식 홈페이지에 공식 또는 AWS 오피셜 샘플 대시보드가 등록되어 있습니다.
*	Grafana Dashboard ID: 20579 또는 23514 (EFA Metrics EKS)
*	방법: 1. 그라파나 웹 콘솔 접속
2. Dashboards ‭->  New ‭$\rightarrow$‬ Import 선택
3. Import via grafana.com 창에 23514 입력 후 Load
4. 데이터 소스를 현재 EKS와 연동된 Prometheus로 지정하면 수동 제작할 필요 없이 즉시 화려한 EFA 트래픽 그래프가 완성됩니다.
