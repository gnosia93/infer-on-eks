# infer-on-eks 

LLM 인프라 구축부터 인공지능 모델 취약점 점검까지: EKS 기반 AI 엔지니어링 실무
EKS(Amazon Elastic Kubernetes Service) 기반의 대규모 분산 추론 환경 구축부터 오픈소스 모델의 성능 및 보안 취약점 평가(Redteaming)까지, AI 인프라와 모델 엔지니어링의 핵심 실무를 다루는 6시간 완성 집중 워크샵입니다.

### 📅 커리큘럼 및 타임라인 (총 6시간 소요) ###
* [Part 1] AI 인프라 기초 (이론 / 1.5H)
  * AI/LLM 워크로드를 위한 클라우드 인프라 아키텍처 이해
    * GPU 자원 효율화: 비싼 GPU가 쉬지 않고 일하게 만들면서도, 사용자가 몰릴 때 자동으로 컴퓨터 대수를 늘리는(Auto-scaling) 구조 설계법
    * 네트워크 병목 해결: 수많은 GPU 컴퓨터들이 서로 초고속으로 데이터를 주고받으며(네트워크 아키텍처) 대형 모델을 나눠서 처리하는 방법
    * 서빙 최적화: 전 세계 사용자가 ChatGPT 같은 서비스를 쓸 때, 답변이 끊기지 않고 실시간으로 빠르게 나가도록 앞단에 로드밸런서(트래픽 분산 장치)와 캐시 시스템을 배치하는 설계도 그리기
  * GPU 자원 할당 및 대규모 서빙 환경의 고충과 해결책

* [Part 2] EKS LeaderWorkerSet (이론 및 실습 / 2.5H)
  * 분산 학습 및 추론을 위한 핵심 오브젝트 LeaderWorkerSet(LWS) 개념 파악
  * EKS 환경에서 LWS를 활용한 다중 노드 GPU 클러스터링 및 대규모 모델 서빙 실습

* [Part 3] 오픈소스 모델 성능 및 취약점 평가 (이론 및 실습 / 2.0H)
  * LLM 평가지표(Benchmark) 및 보안 위협(Jailbreak, PII 유출 등) 이론
  * 자동화 취약점 점검 도구(Promptfoo 등)를 활용한 모델 모의 해킹 및 방어벽 구축 실습

### 🎯 워크샵 핵심 목표 ###
* infer-on-eks 구조를 통해 대규모 언어 모델을 효율적으로 확장(Scaling)하는 방법을 마스터합니다.
* 단순히 인프라를 띄우는 것에 그치지 않고, 서빙되는 오픈소스 모델이 보안적으로 안전한지 직접 해킹하며 검증하는 End-to-End 역량을 확보합니다.
