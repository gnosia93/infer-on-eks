# infer-on-eks 

LLM 인프라 구축부터 인공지능 모델 취약점 점검까지: EKS 기반 AI 엔지니어링 실무
EKS(Amazon Elastic Kubernetes Service) 기반의 대규모 분산 추론 환경 구축부터 오픈소스 모델의 성능 및 보안 취약점 평가(Redteaming)까지, AI 인프라와 모델 엔지니어링의 핵심 실무를 다루는 6시간 완성 집중 워크샵입니다.

### 📅 커리큘럼 및 타임라인 (총 6시간 소요) ###
* [Part 1] AI 인프라 기초 (이론 / 1.5H)
  * AI/LLM 워크로드를 위한 클라우드 인프라 아키텍처 이해
    * GPU 자원 효율화: 비싼 GPU가 쉬지 않고 일하게 만들면서도, 사용자가 몰릴 때 자동으로 컴퓨터 대수를 늘리는(Auto-scaling) 구조 설계법
    * 네트워크 병목 해결: 수많은 GPU 컴퓨터들이 서로 초고속으로 데이터를 주고받으며(네트워크 아키텍처) 대형 모델을 나눠서 처리하는 방법
    * 서빙 최적화: 전 세계 사용자가 ChatGPT 같은 서비스를 쓸 때, 답변이 끊기지 않고 실시간으로 빠르게 나가도록 앞단에 로드밸런서(트래픽 분산 장치)와 캐시 시스템을 배치하는 설계도 그리기
  * GPU 자원 할당 및 대규모 서빙 환경의 고충과 해결책
  ```
  **😭 고충 (Pain Points)
•	"GPU가 너무 비싼데 놀고 있어요" (낮은 자원 효율성)
•	LLM은 사용자가 질문할 때만 잠깐 GPU를 빡세게 쓰고, 생각을 마친 뒤 답변을 출력할 때는 상대적으로 자원을 덜 씁니다. 질문이 안 들어오는 새벽에는 수천만 원짜리 GPU가 그냥 켜진 채 놉니다. 그렇다고 서버를 꺼두면 다음 사용자가 접속했을 때 AI가 켜지는 데 몇 분씩 걸립니다.
•	"모델이 너무 커서 GPU 한 대에 안 들어가요" (메모리 부족, OOM)
•	최신 오픈소스 모델들은 덩치가 너무 커서(예: Llama 3 70B 등) 좋은 GPU 1대(H100 등)의 메모리조차 초과해 버립니다. 서버가 픽픽 쓰러지는 'Out Of Memory(OOM)' 에러와의 전쟁이 시작됩니다.
•	"사용자가 조금만 몰려도 답변이 기어 가요" (트래픽 스파이크 & 서빙 지연)
•	일반 웹사이트는 사용자가 몰리면 텍스트나 이미지만 더 보내주면 되지만, LLM은 한 사람이 긴 질문을 던질 때마다 GPU 연산량이 폭발합니다. 수십 명이 동시에 질문하면 답변 속도가 분당 몇 글자 수준으로 느려집니다.
2. 😎 해결책 (Solutions)
워크샵의 핵심인 **EKS(쿠버네티스)**와 최신 서빙 프레임워크를 도입하면 이 고충들을 멋지게 해결할 수 있습니다.
•	해결책 A: 'LeaderWorkerSet(LWS)'를 통한 분산 서빙 (메모리 부족 해결)
•	모델이 너무 커서 GPU 1대에 안 들어가면, 여러 대의 GPU 컴퓨터를 하나의 팀으로 묶어줍니다. 대장(Leader) 컴퓨터가 조율하고 일꾼(Worker) 컴퓨터들이 모델을 조각조각 나누어 가져가서 연산합니다. 이번 워크샵 실습에서 다룰 핵심 기술입니다.
•	해결책 B: 'vLLM', 'TGI' 같은 고속 추론 엔진 도입 (속도 및 자원 효율 해결)
•	PagedAttention(페이지드 어텐션): 컴퓨터의 가상 메모리 기법을 AI에 도입해, 놀고 있는 GPU 메모리 파편들을 싹 고르게 모아 활용합니다. 이것만 도입해도 GPU 효율이 2~4배 올라가서 더 많은 사용자를 동시에 받습니다.
•	Continuous Batching(연속 배칭): 사용자들이 각자 다른 타이밍에 질문을 던지더라도, AI 엔진이 실시간으로 질문들을 묶어서(Batch) GPU에 한 번에 밀어 넣습니다. GPU가 쉬는 시간을 제로(0)에 가깝게 만듭니다.
•	해결책 C: 쿠버네티스반(EKS)의 KEDA / Karpenter 활용 (비용 절감)
•	사용자가 없는 밤에는 GPU 서버 개수를 자동으로 최소한으로 줄이고(Auto-scaling), 아침에 출근족들이 몰리면 1~2분 만에 GPU 서버를 수십 대로 늘려주는 클라우드 자동화 시스템을 구축합니다.**
  ``` 
* [Part 2] EKS LeaderWorkerSet (이론 및 실습 / 2.5H)
  * 분산 학습 및 추론을 위한 핵심 오브젝트 LeaderWorkerSet(LWS) 개념 파악
  * EKS 환경에서 LWS를 활용한 다중 노드 GPU 클러스터링 및 대규모 모델 서빙 실습

* [Part 3] 오픈소스 모델 성능 및 취약점 평가 (이론 및 실습 / 2.0H)
  * LLM 평가지표(Benchmark) 및 보안 위협(Jailbreak, PII 유출 등) 이론
  * 자동화 취약점 점검 도구(Promptfoo 등)를 활용한 모델 모의 해킹 및 방어벽 구축 실습

### 🎯 워크샵 핵심 목표 ###
* infer-on-eks 구조를 통해 대규모 언어 모델을 효율적으로 확장(Scaling)하는 방법을 마스터합니다.
* 단순히 인프라를 띄우는 것에 그치지 않고, 서빙되는 오픈소스 모델이 보안적으로 안전한지 직접 해킹하며 검증하는 End-to-End 역량을 확보합니다.
